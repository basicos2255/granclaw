# FIX 123.1 - OpenClaw Setup Hardening & Scoped Reauthorization

**Fecha**: 2026-05-05
**Estado**: Completado
**Autor**: Claude

## Resumen Ejecutivo

Corrección y mejora del sistema de setup/pairing de OpenClaw implementado en FIX 123. El sistema ahora utiliza **requisitos granulares por scope** en lugar de un flag global, permite **blocking inteligente** que no afecta queries simples, y **requiere verificación real** antes de marcar OpenClaw como listo.

## Problemas Identificados en FIX 123

| Problema | Impacto |
|----------|---------|
| `openclawRequiresSetup === true` bloquea TODO | Queries simples fallan innecesariamente |
| `/system/mark-openclaw-ready` sin verificación | Sistema puede quedar inconsistente |
| Sin auto-clear cuando OpenClaw funciona | Estado queda "stuck" después de resolver |
| Setup genérico sin distinguir scopes | No se sabe qué permiso específico falta |
| UI solo muestra flag booleano | Usuario no sabe qué hacer específicamente |

## Principios de Diseño

1. **Granularidad**: Cada requisito tiene scope específico (os:open_app, os:install, etc.)
2. **Blocking inteligente**: Queries simples (`simple_question`, `analysis_task`) nunca bloquean
3. **Verificación obligatoria**: No se puede marcar listo sin pasar auth check real
4. **Auto-clear**: Al ejecutar exitosamente un scope, se resuelve su requirement
5. **UI accionable**: Usuario ve qué scopes necesitan autorización

### 1. Tipos Granulares (system-state/types.ts)

```typescript
export type OpenClawScopeKey =
  | 'os:open_app'
  | 'os:install'
  | 'os:filesystem'
  | 'os:browser'
  | 'os:system'
  | 'openclaw:unknown_scope'

export interface OpenClawSetupRequirement {
  id: string
  scopeKey: OpenClawScopeKey
  capabilityKey?: string
  provider: 'openclaw'
  reason: string
  originalError?: string
  status: 'active' | 'resolved'
  createdAt: string
  updatedAt: string
  resolvedAt?: string
}

export interface SystemState {
  // ... existing fields

}
```

### 4. Orchestrator Blocking Inteligente

En `orchestrator/routes.ts`, 4 puntos de blocking actualizados:

```typescript
// Antes (FIX 123)
if (shouldBlockForSetup()) {
  // Bloqueaba siempre
}

// Después (FIX 123.1)
const scopeKey = getScopeFromCapability(capabilityKey)
const isSimpleQuery = intent.kind === 'simple_question' || intent.kind === 'analysis_task'

if (shouldBlockForSetup({ scopeKey, capabilityKey, isSimpleQuery })) {
  // Solo bloquea si:
  // 1. No es query simple
  // 2. Hay requirement activo para este scope
}
```

### 6. Mark Ready Verificado (system-state/routes.ts)

```typescript
// POST /system/mark-openclaw-ready
// Ya no es manual, requiere verificación real

export async function handleMarkOpenClawReady(req, res): Promise<void> {
  // Llama a checkOpenClawAuth internamente
  const authStatus = await checkOpenClawAuth()

  const wsOk = authStatus.ws === 'ok'
  const restOk = authStatus.rest === 'ok'
  const toolsOk = authStatus.tools === 'ok'

  if (wsOk && (restOk || toolsOk)) {
    const resolvedCount = resolveAllRequirements()
    ok(res, {
      success: true,
      verified: true,
      message: 'OpenClaw verified and marked as ready',
      resolvedCount,
      authStatus: { ws, rest, tools }
    })
  } else {
    ok(res, {
      success: false,
      verified: false,
      message: 'Verification failed - OpenClaw not ready',
      authStatus,
      activeRequirements
    })
  }
}
```

```typescript
// Labels legibles
const formatScopeKey = (scopeKey: string): string => {
  const labels: Record<string, string> = {
    'os:open_app': 'Abrir aplicaciones',
    'os:install': 'Instalar aplicaciones',
    'os:filesystem': 'Acceso a archivos',
    'os:browser': 'Control de navegador',
    'os:system': 'Control del sistema',
    'openclaw:unknown_scope': 'Permiso desconocido'
  }
  return labels[scopeKey] || scopeKey
}
```

## Flujo de Blocking

```
┌─────────────────────────────────────────────────────────────┐
│                    Usuario envía acción                      │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│              Clasificar intent                               │
│  simple_question | analysis_task | os_action | ...          │
└─────────────────────────────────────────────────────────────┘
                              │
                   ┌──────────┴──────────┐
                   │                     │
            isSimpleQuery?          OS Action?
                   │                     │
                   ▼                     ▼
┌────────────────────────┐ ┌─────────────────────────────────┐
│ NUNCA BLOQUEA         │ │ Obtener scopeKey de capability   │
│ Ejecutar normalmente  │ │                                  │
└────────────────────────┘ └─────────────────────────────────┘
                                         │
                                         ▼
                           ┌─────────────────────────────────┐
                           │ ¿Hay requirement activo para    │
                           │ este scope?                     │
                           └─────────────────────────────────┘
                                    │           │
                                   NO          YES
                                    │           │
                                    ▼           ▼
                      ┌─────────────────┐ ┌────────────────────┐
                      │ Ejecutar OpenClaw│ │ Bloquear           │
                      │                  │ │ Retornar setup_req │
                      └─────────────────┘ └────────────────────┘
                                │
                       ┌────────┴────────┐
                       │                 │
                    SUCCESS           REAUTH
                       │                 │
                       ▼                 ▼
          ┌────────────────────┐ ┌────────────────────────────┐
          │ resolveRequirement │ │ addSetupRequirement        │
          │ para este scope    │ │ con scopeKey detectado     │
          └────────────────────┘ └────────────────────────────┘
```


### Verificar y marcar como listo
```bash
curl -X POST http://localhost:3001/system/mark-openclaw-ready -H "Authorization: Bearer $TOKEN"
# Retorna verified: true/false, resolvedCount
```
